📊 統計頁大升級：盲門、顏色分佈、冷熱號
2026-05-11

大幅強化 stats.html 嘅分析功能：

  • 盲門：定義做「最近 180 期從未出現過嘅攪出號碼」，用灰色波波顯示，等大家可以留意極度冷門嘅號碼。
  • 顏色分佈：跟返官方號碼顏色規則（紅/藍/綠），畫咗個圓餅圖，一眼睇晒邊種顏色出得多。
  • 熱冷號定義：熱號 = 最近 20 期出現次數最多嘅前 10 個號碼；冷號 = 出現最少嘅前 10 個。
  • 另外仲加咗和值直方圖、區間分佈、尾數分佈等圖表。

所有圖表都係每次 refresh 頁面時直接讀 Google Sheets 嘅最新數據，唔使人手更新。

🐛 修正首頁期號同 NaN 問題
2026-05-10

之前有用戶反映主頁顯示「26/047 NaN,9,17,...」，查完發現：

  • Google Sheets 其實已經有期號欄位，但前端 loadCsvData() 冇讀嗰一欄，反而用 47 - index 嚟自己計期號，搞到錯晒。
  • 修正方法：擴大讀取範圍到 A:H 列，正確解析期號同 6+1 個號碼。
  • 同時改埋 renderSingle 入面最新一期同最近 5 期嘅顯示方式。

而家主頁正確顯示「26/049 9,17,26,41,42,47 + 特別號 8」，冇晒 NaN。

📁 數據儲存嘅黑暗歷史：Google Sheets API 同 Service Account 嘅搏鬥
2026-05-11

原本以為揾到數據源就搞掂，點知寫入 Google Sheets 先係另一個地獄。記錄低啲踩坑位，俾自己同將來嘅人睇：

  • Service Account 權限：要喺 Google Cloud Console 開一個 Service Account，下載 JSON 金鑰，然後將個 email 加入 Google Sheets 嘅分享權限（編輯者）。試咗好多次先發現原來係要 share 俾 xxx@xxx.iam.gserviceaccount.com，唔係俾自己 email。
  • GitHub Secrets 格式：個 JSON 金鑰入面有換行同雙引號，直接貼入 GitHub Secrets 會爛。解決方法係將成個 JSON 壓成一行（或者 base64），再喺 Actions 入面 decode。最後我揀咗直接貼 raw JSON，但要用三引號包住喺 workflow 嘅 env 度。
  • gspread 版本問題：actions/setup-python 預裝嘅 gspread 太舊，oauth2client 又 deprecated。結果要 pip install 指定版本：gspread==5.12.0 同 google-auth-oauthlib。
  • 讀取同寫入嘅 quota：免費版 Google Sheets API 每日有讀寫配額，好彩我哋一日只寫一次，讀取只係用戶 refresh 嗰陣先會發生，暫時夠用。
  • Apps Script 中伏：最初想用 Google Apps Script 做 proxy，慳返 API key。但 Apps Script 嘅 UrlFetch 成日 timeout，日誌又難 debug，最後放棄，直接改用 Google Sheets API v4 配 API key 唯讀，寫入就用 Service Account。
  • 寫入時嘅亂碼：有期號包含 '/'，寫入 sheets 時會自動被轉成日期格式（例如 26/049 變 2026-04-09）。解決方法：寫入前將期號加一個前置單引號 '26/049，強制當作文本。

而家個流程穩定咗：
Python 用 gspread + Service Account JSON（放喺 GitHub Secret）→ 每日一次 insert_row 到第二行（最頂係最新一期）→ 前端直接用 API key 讀取 A:H 範圍。雖然中間撞板撞到七彩，但總算行得通。

⏰ 自動化架構：GitHub Actions + Netlify
2026-05-09

為咗確保每日數據自動更新，我哋設定咗：

  • GitHub Actions：每日香港時間 22:03 執行 Python 爬蟲，將最新一期結果寫入 Google Sheets。
  • Google Sheets API：前端的 index.html 同 stats.html 直接透過 API 讀取試算表，確保用戶永遠睇到最新數據。
  • Netlify 自動部署：每次 push 到 GitHub，Netlify 會自動 build 同 deploy，唔使手動攪。
  • 另外仲保留手動觸發（workflow_dispatch），方便即時測試。

呢個架構令到維護成本極低，而且完全免費。

🧠 預測算法核心：加權評分 + 組合過濾
2026-05-08

個 AI 預測唔係亂抽，背後用咗幾層統計模型：

  • 加權評分：越近期嘅號碼權重越高（指數衰減），同時考慮熱號、冷號、避免重覆上期號碼。
  • 組合生成：單式由最高分嘅 20 個號碼入面隨機抽 6 個（但係用固定種子，可重現）。
  • 複式：直接拎最高分嗰 N 個號碼 (N 可選 7~20)。
  • 膽拖：先揀最高分嘅膽碼，再喺剩餘號碼入面揀最高分嘅拖碼。

目前嘅參數（回顧期數 8、衰減 lambda 0.35、冷號門檻 6 期、熱號加成 1.2）仲可以透過真實命中率回測再 fine-tune。
